➣ You must installed NodeJs on server. And if you don't have please install it before move ahed. Click

If you are using a non-dedicated (shared) server that does not support resource-intensive processes like building applications, it is recommended to build the project on your local machine. After building locally, upload the build files to the server and deploy the application using PM2 on Apache or Nginx by following the steps outlined below.

➣ If you're using a Cloud/VPS account, extract the grabit-react.zip file to your hosting folder (e.g., public_html). Otherwise, upload the .next and public folders, along with the package.json and package-lock.json files to the shared server's public_html folder, and remember to update your domain in the .htaccess file.

➣ Get Login into your account and Goto "Website - your site > Advanced > SSH Access" and access SSH terminal.

➣ Open SSH terminal and Goto domain's project root folder

  • Set NEXT_PUBLIC_DOMAIN Enviroment variable in .env.production
    • NEXT_PUBLIC_DOMAIN="https://yourdomain.com"
  • Set NEXT_PUBLIC_PATH Enviroment variable in .env.production file if you want to configure with subfolder
    • NEXT_PUBLIC_PATH="/subfolder/"
  • Uncomment below line in next.config.mjs file if you want to configure with subfolder
    • basePath: process.env.NEXT_PUBLIC_PATH,
  • Run commands to install dependency and create build
    (If you're using shared hosting, build your project locally and upload the .next, public folders, along with the package.json and package-lock.json, .htaccess files to your server.)
    • npm i
    • npm run build
  • Run command to deploy build with apache/nginx on server
    (For share hosting, use only first four steps)
    • npm install pm2 -g
    • pm2 delete "grabit-next"
    • pm2 start npm --name "grabit-next" -- run start
    • pm2 save
    • configure server with runing port i.e 3000
      • Apache (/etc/apache2/sites-available/000-default.conf) 
        <VirtualHost *:80>
	ServerName yourdomain.com

	ProxyPreserveHost On
	
	# comment out below two line if you want to configure with subfolder
	ProxyPass / http://localhost:3000/
	ProxyPassReverse / http://localhost:3000/

	# uncomment this section if you want to configure with subfolder
	# ProxyPass /subfolder http://127.0.0.1:3000/
	# ProxyPassReverse /subfolder http://127.0.0.1:3000/
</VirtualHost>
      • Nginx (/etc/nginx/sites-available/default) 
        server {
	listen 80;
	server_name yourdomain.com;

	# comment out below section if you want to configure with subfolder
	location / {
		proxy_pass http://localhost:3000/;
		proxy_http_version 1.1;
		proxy_set_header Upgrade \$http_upgrade;
		proxy_set_header Connection 'upgrade';
		proxy_set_header Host \$host;
		proxy_cache_bypass \$http_upgrade;
	}

	# uncomment this section if you want to configure with subfolder
	# location /subfolder/ {
	#     proxy_pass http://localhost:3000/;
	#     proxy_http_version 1.1;
	#     proxy_set_header Upgrade $http_upgrade;
	#     proxy_set_header Connection 'upgrade';
	#     proxy_set_header Host $host;
	#     proxy_cache_bypass $http_upgrade;
	# }
}
  • Or Run script to install npm dependency, build and server configure
    (Proceed if your server supports full deployment)
    • bash deploy.sh
